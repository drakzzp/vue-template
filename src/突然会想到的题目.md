useState 用于在函数组件中添加状态，返回一个状态可以改变状态的函数，改变状态可以改变试图
useRef：创建一个可变的 ref 对象，其 .current 属性在组件的整个生命周期内保持不变。常用于访问 DOM 元素、保存可变数据（不会触发重新渲染）或在函数组件中模拟实例变量。

useCallback：用于缓存函数，返回一个 memoized 回调函数。只有当依赖数组中的值发生变化时，才会返回新的函数，否则返回缓存的函数。常用于将回调函数传递给子组件，以避免不必要的子组件重新渲染。

useMemo：用于缓存计算结果，它接收一个回调函数和依赖数组。只有当依赖数组中的值发生变化时，才会重新计算回调函数的结果，否则返回缓存的值。这有助于避免在每次渲染时都进行昂贵的计算。

useReducer：类似于 Redux 中的 reducer，适用于管理复杂状态逻辑，尤其是当状态更新依赖于之前的状态时。它接收一个 reducer 函数和初始状态，返回当前状态和一个 dispatch 函数，通过 dispatch 函数触发 reducer 来更新状态。

useEffect：用于处理副作用操作，例如数据获取、订阅或手动更改 DOM。它接收一个回调函数，该函数会在组件渲染和更新后执行（默认情况下）。可以通过返回一个清理函数来处理副作用的清理，比如取消订阅。

useContext：用于在组件树中共享数据，避免通过 props 层层传递数据。先创建一个 Context 对象，然后在需要共享数据的组件中使用 useContext 来获取该 Context 的值。

useLayoutEffect：与 useEffect 类似，但它会在 DOM 更新后、浏览器绘制之前同步执行。适用于需要读取或操作 DOM 布局，并同步更新状态的场景，避免页面闪烁等问题。

Monorepo 工程化见解及常见目录结构
apps 目录
用途：存放各个具体的应用程序。这些应用通常是面向用户的产品，如 Web 应用、移动应用、桌面应用等。每个应用都可以独立运行、部署。

packages 目录
用途：用于存放可复用的模块、库或工具集。这些包可以被apps目录下的应用以及其他packages中的模块依赖。

config 目录
用途：集中存放整个 Monorepo 的配置文件，包括构建配置、开发环境配置、测试配置等。这样可以统一管理和维护配置，避免在各个应用和模块中重复设置。

tools 目录
用途：存放各种开发工具脚本，用于辅助项目的开发、测试、部署等流程。这些脚本可以是自定义的命令行工具、自动化脚本等。

docs 目录
用途：存放项目的文档，包括项目介绍、开发指南、API 文档、架构设计文档等。良好的文档有助于新成员快速上手，也方便团队成员之间的沟通和协作。

react16 和 react18的区别
增加了createRoot api, support concurrentMode
然后setState自动批量更新，
useId 生成稳定唯一 ID，解决 SSR 与客户端 hydrate 不一致问题, 
useTransition 标记非紧急更新，延迟渲染非关键内容，保证主线程流畅, 
useDeferredValue 延迟计算非关键值，类似防抖，降低高负载渲染阻塞
useSyncExternalStore 规范订阅外部状态库（如 Redux、MobX），适配并发渲染，避免状态不一致问题
useInsertionEffect 专为样式库设计的插入样式副作用 Hook
