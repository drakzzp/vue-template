有没有做兼容 umd 兼容

UMD 原生内置了 CommonJS、AMD、浏览器全局变量三种模块规范的兼容逻辑，所以我们平台通过 script 动态引入 CDN 上的 umd 组件包时，组件会自动挂载到 window 供画布调用。但 UMD 仅解决模块导入规范问题，无法兼容低版本浏览器 ES6 语法，因此组件工厂打包流程中会集成 Babel 将代码转译为 ES5，并按需注入 polyfill，同时统一规范每个组件挂载到 window 的全局变量名，解决多组件命名冲突问题，实现全场景兼容。

websocket比如连接中断了要怎么办

针对跨屏联动 WebSocket 断连问题，我们四层容错处理：
实现 3 秒一次心跳检测，快速识别网络断开，避免长时间无效连接；
断开后采用指数退避自动重连，延时逐步递增并设置上限，防止高频重连压垮服务；
断连期间缓存所有用户联动操作，重连成功后批量补发，保证多终端数据同步不丢失；
页面销毁时主动关闭 socket 并清空定时器，同时增加业务降级，长时间重连失败则关闭跨屏联动，保障大屏基础可视化功能可用。

有没有做过离线
第一类是跨屏联动 WebSocket 临时离线处理，断网时缓存用户所有联动操作至内存队列，网络恢复重连后批量补发，保证多终端状态同步，同时限制队列长度避免性能问题；
第二类是整套大屏完整离线可用方案，一是支持导出离线静态资源包，打包依赖组件、静态数据，无网络本地直接打开；二是基于 PWA+IndexedDB 实现浏览器缓存离线，持久化大屏配置与离线操作，联网后同步数据。离线场景下会自动降级关闭跨屏联动功能，保障基础可视化查看不受影响，适配油田厂区无外网的业务场景。

问了一个登录鉴权
用户访问：用户尝试访问某个应用系统（如系统 A），但此时用户尚未登录。系统 A 检测到用户未登录，将用户重定向到统一认证中心（SSO Server）。
认证过程：统一认证中心验证用户身份，这可能通过用户名密码、第三方认证（如微信、QQ 登录）等方式。若用户首次登录，需在认证中心进行注册和认证。认证成功后，认证中心会为用户生成一个票据（Ticket），这是用户身份的凭证。
票据传递：认证中心将包含票据的重定向地址返回给系统 A。系统 A 收到后，携带票据向认证中心验证票据的有效性。
访问授权：认证中心验证票据无误后，告知系统 A 该用户已通过认证，系统 A 确认后允许用户访问。同时，在用户后续访问其他应用系统（如系统 B）时，系统 B 同样检测到用户未登录，重定向到认证中心。认证中心发现用户已登录，直接返回包含有效票据的重定向地址给系统 B，系统 B 验证票据后授权用户访问。

SSO = Single Sign-On 单点登录
不跨顶级域名 后端存sessionId在redis/数据库里，前端cookie带上这个sessionId
通过请求携带的 sessionId，去后端会话存储（Redis / 数据库 / 内存）查找对应的有效会话数据

跨顶级域名

浏览器通过 Cookie（默认 connect.sid）携带 sessionId；
express-session 中间件读取 Cookie 里的 sid，用 secret 校验签名防篡改；
用 sid 作为 key 查询外部会话存储（Redis / 数据库）读取会话数据，同时校验过期；
校验通过后挂载 req.session，通过判断 req.session 是否存在登录信息确认登录状态；
可手动取出 req.session.id，直接调用存储 API 做额外校验。


轻量同域 SSO：搭建 Node 统一认证中心，设置主域 Cookie / 全局密钥 JWT，子应用跳转统一登录、共享身份凭证
标准跨域 SSO：用 Node 搭建 OIDC IdP（oidc-provider）或对接成熟 OIDC 服务，走 OAuth2 授权码模式下发 ID Token，Node 子服务校验标准 JWT 身份凭证实现跨域单点登录
核心：统一认证中心 + 标准身份凭证（OIDC JWT）+ 跳转授权模式，区别于单系统本地 JWT/Session 鉴权
